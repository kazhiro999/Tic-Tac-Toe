# デザインレビューフロー

## 基本的なデザインレビューフロー

1. #kintone-design-reviewにkintone Design Teamメンション宛に投稿してレビュー促す
1. 詳しく話したければ朝会へレビュー出すor slackでコメントを募集する
1. sakitoさんのOKがでる（:ok_designのリアクションをもらう）
1. PMにデザインプロトを連絡

## レビューフローの意図

- kintone Design Team内で潰せる箇所は事前に潰しておくため
  - kintone的観点、実装的観点、kDS的、ライティング的な観点などなど…
- デザインの意思決定のやりとりをテキストで残しておきたい
  - Figma上でコメントしてもいいが、コメント消さないなど
- デザイン責任者が目を通してあって、OKが出ている状態でリリースされたい
