# モバイルビュー

## モバイルビューをPCビューで見る方法

- URLを変える（ドメインの後にk/m/をつける）。
  - https://xxx.cybozu.com/k/m/
- ブラウザの開発者ツールのモバイルモードで閲覧する。

## モバイル V2 の経緯

- 2019年にモバイルビューのアップデートがあった。そのアップデート前と後をそれぞれ「モバイルv1」「モバイルv2」と呼び分けている。
- 当時kintoneモバイルの社内外の評価が悪かった。
  - アンケートでも評価が低かった。
  - 営業の商談でもモバイルの紹介を避けられてきた。
- [kintone モバイルの Revival(pdf)](https://bozuman.cybozu.com/k/api/space/blob/download.do/-/2019-01-22-%E3%83%A2%E3%83%9[…]%AA%AC%E6%98%8E%E4%BC%9A-004.pdf?hash=61C51&id=1466974&.pdf)

## ネイティブアプリと Web ビュー

- 多くの画面がWebビューなので、基本的にはiOSでもAndroidでも同じ機能を担保している。
- ネイティブアプリの部分は、ログイン画面等の認証画面（他にもあるかも）。
- ネイティブアプリではなく、Webビューを採用しているのは実装都合（APIの仕様など）。

## モバイルのデザインガイドライン

- iOSとAndroidどちらのガイドラインにも明確に準拠していないと思われる。
- そもそもWebビューなので、Web表示だと捉えている。
- ページ遷移の考え方はWebに従う。
- デザインは必ずしもPCに合わせていない。モバイル独自のコンポーネントもある。

## ページの構成

- すべてのページは以下の3つのペインで構成されている。
  - 左ペイン（メインメニュー）
  - 中央ペイン（コンテンツ）
  - 右ペイン（通知）

## モバイルビューの機能と PC ビューの機能

- モバイルビュー特有のユースケースを考慮するよりも、モバイルビューの機能とPCビューの機能を揃えることが多い。
