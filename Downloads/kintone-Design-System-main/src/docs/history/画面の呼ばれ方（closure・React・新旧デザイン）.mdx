# 画面の呼ばれ方（Closure・React・新旧デザイン）

kintoneの画面はフロントエンドの技術とデザインの新旧で分けて呼ばれることがあります。
呼ばれ方と区分を説明します。

<Notes variant={VARIANTS.WARNING}>
  {`この画面の呼び分けは将来的にはやめたいです。
  歴史的経緯によりこのように区分され呼び分けられていますが、「常に変化するデザイン」を理想としてkintoneの画面デザインがこのような区分がされないことが理想です。`}
</Notes>

## 図の説明

- フロントエンドのスタックとデザインの新旧で分けています（４象限）。
  - argoデザインというkintoneのデザインリニューアルをはさんで以後が「新デザイン」、以前が「旧デザイン」にわかれています。
  - 新デザインをさらに発展させた「新新デザイン」と呼ばれるものもあります。詳しくは下記で説明します。
- Closure × 新デザイン
  - 現在のkintoneの画面の大多数がここにあたります。
- Closure × 旧デザイン
  - kintoneのシステム画面系のページのほとんどが該当します。
  - 例：個人設定ページ、CSV読み込みページ
- React × 旧デザイン
  - Mira（見た目を変えずにリリースするプロジェクト）チームが実装中の範囲です。
  - kintoneのシステム画面系のページの一部が該当します。
- React × 新デザイン
  - ヘッダーとグローバルナビゲーション
    - React化された画面は、React化されたものに実装が置き換わっています。
    - 今後、Closureの画面でも、実装が置き換わる予定です。
- 枠外（React × 新新デザイン）
  - アプリ設定画面（Reactoneと呼ばれていたもの）が該当します。
  - 新デザインに比べ一部のデザインを修正しているため、新新デザインと呼ばれることもあります。
  - 明確な名前はありません。

## 用語

| 用語          | 説明                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| argo デザイン | kintone は 2016 年ごろに一度デザインの大幅リニューアルをしています。（参考：[kintone 新デザイン コンセプト](https://kintone.cybozu.co.jp/support/newdesign.html)）<br/>その際にアメリカのデザイン会社「argodesign」に委託していたため、リニューアル後のデザインやその原型を「argo デザイン」と呼ぶ場合があります。                                                                                                      |
| AppShell      | フロリアのチーム名のひとつです。 <br/>ヘッダーとグローバルナビゲーションを React 化しているチームです。<br/>ヘッダー・グローバルナビゲーションから表示されるポップアップやダイアログも実装範囲に含まれています。                                                                                                                                                                                                        |
| Reactone      | フロリアのチーム名のひとつです。<br/> もともとは kintone を React 化していこうというプロジェクトのひとつで、アプリ設定の設定タブの画面のデザインリニューアルと React 化を同時に行っていました。<br/>その後フロリアプロジェクトにつながり、いまはフロリアの 1 チームの名前となっています。<br/>以上の経緯により、最初に React 化されたアプリ設定の設定タブの各画面をまとめて「 Reactone 画面」と呼ばれることがあります。 |
