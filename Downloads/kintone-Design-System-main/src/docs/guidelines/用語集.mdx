# 用語集

## 用語集で解決したい問題

### 対 kintone ユーザー

- 同じ意味なのに違う言葉が使われていると、ユーザーが迷う。誤認識したり、操作に迷ったりしてしまう可能性がある。
- そもそも名前がついていないと説明しにくい

### 対 kintone 開発者（kintone Design Team）

- 開発に関わるひとでも同じ機能・パーツを指す時に違う言葉を使っているとコミュニケーションの齟齬につながる
- デザイン・アクセシビリティ等の考慮事項、意思決定のストック（同じ言葉が使われると検索にひっかかるようにできる）
- 用語が定義されていないと、何を意味しているのかわからない時がある。誤認識することがある（翻訳の時に）、訳語がブレる
- 文言やヘルプの表現を考えるときに、似たような言葉がいくつかある場合、どちらを選択するか毎回迷う。迷う時間を減らしたい
- そこになければ、まだこれから検討が必要だと認識できる
- そもそも名前がついていないと説明しにくい

## 用語集の対象読者

- kintoneライター
- kintone日英翻訳者
- ヘルプの繁体字、簡体字など社内翻訳者＋外注先の翻訳会社（ツールから参照）
- kintone Design Team

## 用語集の活用方法

- 文言検討する時
- ヘルプを書く時
- 翻訳する時
- kintone Design Team内で機能についてのコミュニケーションをする時

## どの用語までサポートするか

kintoneの機能のみを対象とする。<br/>
画面の呼び方や、コンポーネントの呼び方などkintoneのパーツなども対象とする。<br/>
DXチームが担当しているような機能群は対象外。<br/>
記号の使い方は対象外。（記号については[日本語の記号の用法と英語文言での扱い方](?path=/docs/guidelines-日本語の記号の用法と英語文言での扱い方--docs)を参照）
