# Voice&Tone ガイドライン (for EN)

## Kintone's voice and tone

- 💓 Caring
- 🌈 Respectful
- 💪 Trustworthy
- 🛠️ Practical

## 💓 Caring（面倒見が良い）

Kintone is inclusive and easy to use by any users. Our basic attitude is thoughtful and supportive; but we're motivational at the right moment. We empower users to create their own business systems no matter their IT skills or roles. We know some users are new to system development, and we always have these users in mind; we focus on alleviating their concerns that may arise when building business systems.

>kintoneは、どんな人にも使いやすいインクルーシブな製品です。ユーザーを思いやり、サポートしていくことを基本姿勢としていますが、やるべきときには積極的に提案を行います。部署や立場、ITスキルに関わらず、どんな人でも自分の仕事にぴったりの業務システムを作れるよう全力で応援します。ユーザーの中にはシステム開発の経験がない人がいることもよくわかっており、いつだってその人達を念頭に、システム開発時にユーザーが感じうる不安を解消することに注力します。

**Write in** warm, supportive, and empathetic tone. Show users we understand how they feel. We want to guide and support them through difficulties.

>トーン：温かい、思いやりのある、共感的
>
>ユーザーの気持ちを理解していることを示すような口調で。ユーザーが壁を乗り越えられるよう案内・サポートする姿勢で。

### When we need to be more caring（Caringをより強く出す場面）

- In tutorials, introductory paragraphs, tooltips, information messagesℹ️, and success messages✅
- When something requires actions from the user
- When there are multiple options the user can choose from
- When there are opportunities to give tips and best practices

>- チュートリアル、導入文、ツールチップ、インフォメーションメッセージℹ️、成功メッセージ✅
>- ユーザーが何かアクションを取る必要がある場面
>- ユーザーの選択肢が複数ある場面
>- ヒントやベストプラクティスを紹介するとき

### When we need to be less caring（Caringをより弱く出す場面）

- When the user is confident & skilled enough to proceed on their own
- When the user is self-standing (Ex. power users, IT members, .com admin users)

>- ユーザーに自信または十分なスキルがあり、自力で問題なく操作を進められる場合
>- 助けが不要なユーザーである場合（例：パワーユーザー、IT部門の人、.com共通管理者）

## 🌈 Respectful（尊重）

Kintone listens carefully to each user's needs. We're proud of our flexibility to handle any type of business. We know each user has different needs and requirements, and we always respect them. We're always present, but not pushy; users' choice is our priority.

>kintoneは、ユーザー一人ひとりのニーズにじっくりと耳を傾けます。あらゆる業種のあらゆる仕事に対応できる――そんな高い柔軟性が自慢です。ユーザーによってニーズや要件は様々であることを理解しており、いつでも個々の考えや希望を尊重します。いつも近くにいるけれど、押し付けることはしません。ユーザーの選択が最優先です。

**Write in** humble, courteous, and neutral tone. Be open to any option available. Never be pushy or forceful.

>トーン：謙虚、礼儀正しい、中立的
>
>どんな選択肢にもオープンで。決して押し付けたり、強制したりしない。

### When we need to be more respectful（Respectfulをより強く出す場面）

- In modal dialogs, buttons, setting options, information messagesℹ️, and error messages🚫
- When something requires actions from the user
- When there are multiple options the user can choose from

>- モーダルダイアログ、ボタン、設定項目、インフォメーションメッセージℹ️、エラーメッセージ🚫
>- ユーザーが何かアクションを取る必要がある場面
>- ユーザーの選択肢が複数ある場面

### When we need to be less respectful（Respectfulをより弱く出す場面）

- When the user is trying to perform a destructive action
- When the user's action may result in a critical error
- When there is a possibility that the user may not be able to use the service comfortably if they continue with what they're doing

>- ユーザーが破壊的な操作（削除などやり直しが利かない操作）を行おうとしている場合
>- ユーザーの行動によって重大なエラーが発生する可能性がある場合
>- このまま続けるとkintoneを快適に利用できなくなる可能性がある場合

## 💪 Trustworthy（信頼できる）

Kintone has an extensive experience in supporting customers with their business system development. We know where the user's difficulties are, and we can give quick and thorough answers, guidance, and instructions at the right time. We act sincerely and honestly when dealing with difficult situations because we genuinely care about the user's success.

>kintoneは、これまで沢山のユーザーの業務システム開発をサポートしてきました。経験豊富で、ユーザーがつまづきやすい部分についてよく知っているからこそ、適切なタイミングで丁寧に答えを教えたり、案内したり、素早く手を差し伸べたりすることができます。本気でユーザーの成功を考えているから、難しい状況に直面したときには誠実に、正直に行動します。

**Write in** professional and reliable tone. Speak with honesty and sincerity. Explain in a bold and clear tone if there's something the user needs to know.

>トーン：プロフェッショナル、頼りになる
>
>実直で、誠実な口調で。ユーザーが知るべきことがある場合は、はっきりと明示的に伝える。

### When we need to be more trustworthy（Trustworthyをより強く出す場面）

- In advanced settings (i.e. settings configured by IT members, developers, and .com admin users), warning messages⚠️, and error messages🚫
- When the user is having a issue and needs help
- When the user's action may result in a critical error
- When the target user is an IT professional or has high IT skills (Ex. IT members, developers)

>- 高度な設定（IT部門、開発者、.com共通管理者向けの設定）、警告メッセージ⚠️、エラーメッセージ🚫
>- ユーザーが問題に直面し、助けを必要としているとき
>- ユーザーの行動によって重大なエラーが発生する可能性がある場合
>- ユーザーがITの専門家である場合や、ユーザーに高いITの知識がある場合（例：IT部門の人、開発者）

### When we need to be less trustworthy （Trustworthyをより弱く出す場面）

- In tutorials, success messages✅, and tooltips
- When the user is less confident & not skilled enough to proceed on their own

>- チュートリアル、成功メッセージ✅、ツールチップ
>- ユーザーがスキル不足などで自信がなく、自力で操作を進めることができない場合

## 🛠️ Practical（実践的）

Kintone is down-to-earth; we know building and managing business systems can easily get complex and we won't let that happen. We are plainspoken, and we simply tell users what they need to know, without unnecessary decoration. We know that having just about the right amount of information is better than having everything.

>kintoneは堅実で、地に足がついた製品です。業務システムの構築や管理は簡単ではなく、複雑になりがちだということを理解した上で、そうならないよう全力を尽くします。不要な修飾語は使わず、平易な言葉で、ユーザーに必要な情報をシンプルに伝えます。必要なものを必要な分だけ得ることのほうが、何もかも与えられるよりも良いとわかっているからです。

**Write with** clarity and briefness. Get straight to the point. Keep the sentence structure simple.

>トーン：明快に、簡潔に
>
>文構造をシンプルに。要点を端的に伝える。

### When we need to be more practical（Practicalをより強く出す場面）

- In warning messages⚠️ and error messages🚫
- When the user's action may result in a critical error
- When the user is an IT professional or has high IT skills (Ex. IT members, developers)

>- 警告メッセージ⚠️、エラーメッセージ🚫
>- ユーザーの行動が重大なエラーにつながる可能性がある場合
>- ユーザーがITの専門家である場合や、ユーザーに高いITの知識がある場合（例：IT部門の人、開発者）

### When we need to be less practical（Practicalをより弱く出す場面）

- In tutorials, introductory paragraphs, and success messages✅
- While introducing new experiences
- When the user is less confident & not skilled enough to proceed on their own
- When there are opportunities to share with users positive feelings such as success and joy

>- チュートリアル、導入文、成功メッセージ✅
>- ユーザーがスキル不足などで自信がなく、自力で操作を進めることができない場合
>- 成功や喜びなど、ポジティブな感情をユーザーと共有できる機会があるとき
