# アイコンについて

## アイコンのバリエーション

### 旧デザインのアイコン

Argo デザインにリニューアルされる前のアイコン。
旧デザインの画面で使用されていて、立体感があったり、暖色系のグレーが使われているのが特徴。

### Argo デザインのアイコン

Argo デザインにリニューアルされた時に作成されたアイコン。
角丸がなくソリッドな形で、塗り潰しのスタイルが基本。
例外として、グローバルヘッダーのハンバーガーメニュー内のアイコンはラインスタイルなのと、投稿のいいねボタンやレコード追加ボタンはグレーが 2 色になっている。

### フィールドのアイコン

Argo デザインへのリニューアル後にアプリ作成画面をリニューアルした時に作成されたアイコン。
グレーが何色か使われていて、ディティールがやや細かい。

---

## アイコンのデータが完全でない理由

- Argo デザインにリニューアルした当時のデザインデータが完全な形では残っておらず、書き出されたラスターデータしか残っていないものがあった
- kintone のデザインデータが Photoshop→Sketch→Figma というツールの変遷を経ており、データを移行するタイミングで欠落や意図しない変更が発生している
- アイコン自体の品質の問題で、利用するサイズに対してデザインを最適化する必要が生じるなどしてバリエーションが作られてしまう
- Argo デザインのドキュメントが運用できていないかつ、デザインの体制も不安定だったため作成者によってテイストが変わってしまっている

---

## IconButton、IconTextButton、TextButton の使い分け

### IconButton

- 設定（歯車）、検索（虫眼鏡）のような他のサービスでも一般的に使われるアイコン
- ツールバーなどの限られたスペースにボタンを配置する必要がある
  ※ kintone 独自の概念を無理やりアイコンにはしない。その場合はメニュー内に表示するなど、テキストで表現できる場所がないか検討する
- 位置などのコンテキストで機能が説明できる（aria-label などは適切に設定すること）

### IconTextButton

- アイコンでボタンの意味をより明確にできる場合
- アイコンだけでは表現しづらいが、テキストをつければ成り立つ場合

### TextButton

- **kintone の独自の概念**で、アイコンをつけても理解しやすさに寄与しない場合
- 保存、キャンセル、などテキストボタンとして一般的に広く使われている場合（この場合はアイコンをつけることが逆に理解の妨げになる）

画面内に占める面積は必然的に IconText > Text > Icon となるので、IconText を多用すると画面が乱雑になりがち。重要度のヒエラルキーなどのバランスを見てそれぞれを使い分ける。
