# アクセシビリティデザインガイドライン

デザイン検討時にアクセシビリティの観点で考慮してほしいことをまとめます。

各ガイドラインは［must］と［should］の2つの重要度にわけられます。

- must
  - かならず守ってほしい
  - なんらかの理由で守れない場合は要相談
    - kintone Design朝会で相談する
    - [#kintone-design-review](https://cybozu.slack.com/archives/C053W5SHJCS)で、@kintone-design-system-teamにメンションする
- should
  - 可能な場合検討する
  - 検討するかどうかはユースケースごとに判断
  - 可能なら検討できると良いユースケース（一例）
    - 頻繁に使用することが想定される場合
    - 重要なアクションを引き起こす場合（削除など）
    - 操作できないとタスクが完了できない場合

前提として、［must］については、現実的に可能なガイドラインのみ記載します。

## 知覚可能

### 色だけで情報を伝えない

#### コンセプト

誰に：

- 色覚多様性
- 視力が低下している人

なにを：

- 色が区別できなかったとしても情報を理解できる

#### 検討事項

- [Spectrum](https://chrome.google.com/webstore/detail/spectrum/ofclemegkcmilinpcimpjkfhjfgmhieb?hl=ja)などのツールを使って、白黒やいろいろな色覚で情報が伝わるかを検証する
- 色以外の感覚特性を併用した表現にする
  - 色＋テキスト
  - 色＋形・記号

#### 参考情報

- [色だけで情報を伝えない | サイボウズ共通アクセシビリティガイドライン](https://bozuman.cybozu.com/k/34242/show#record=27)

## 操作可能

### ［should］ 最も重要なアクションを引き起こすボタンのタッチサイズは 44 x 44px 以上にする

#### コンセプト

誰に：

- タッチユーザー
- 細かい作業が苦手な人
- 細かい作業が困難の状況（電車内など）

なにを：

- ボタンが押しやすい
- 操作ミスが少ない

#### 検討事項

最も重要なアクションを引き起こすボタンのタッチサイズは44 x 44px以上にします。<br/>
例：［保存］ボタン、Dialogの ［OK」ボタン、取り消すことのできない［削除］ボタンなど。

#### 参考情報

- [達成基準 2.5.5: ターゲットのサイズを理解する](https://waic.jp/translations/WCAG21/Understanding/target-size.html)
