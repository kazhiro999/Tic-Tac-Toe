# 文言で気をつけていること

- 常用漢字表にない読み方は使用しない
- 文章のあとに補足説明を付ける場合、句点はかっこのあとにつける<br/>
  例：削除したアプリやスペースを、削除時点の状態に復旧することができます（一部のアプリ設定を除く）。
- 漢字とひらがなを適切に使い分ける<br/>
  モバイルなどスペースが足りない場面では漢字を多めに使用することもある<br/>
  管理者系画面がかちっとした文章にすることが多い
- ひとつの説明文の中で、一度使った単語を別の言葉に置き換えない<br/>
  初出の言葉を出してユーザーに疑問・ひっかかりを持たせない<br/>
  例：ユーザーは～と使っている説明文の中に「利用者」を混同させる
- 敬語を使いすぎない<br/>
  使っている例：「ご確認ください」 「ご用意しています」<br/>
  いらいらを和らげたいとき、手間かけるときには丁寧な敬語を使う場合がある
- 句点（。）を省略する場合
  - 見出し
  - Checkbox、Radio Button
  - 不完全文（完全文とは：文が完結している完全な文のこと）
- 「〜〜できます」と「〜〜可能です」の使い分け<br/>
  「〜〜できます」の方が多い <br/>
  「〜〜可能です」を避けることはしないが、前後の文章や、読んだときのリズムや、類似の既存文言などを参考に使い分けている。<br/>
  「やろうと思えばできますよ」要素を出したいときは「～～可能です」の方を使う<br/>
- 数字と英単語の前後に半角スペースを挿入しない<br/>
  和欧混植の文章内の英単語と、数字の前後には半角スペースは入れずに詰める。
- 記述記号は全角を使用し、前後にスペースは入れない<br/>
  記述記号は基本的に全角を使用し記号の前後にスペースは挿入しない。英数字とあわせて表記する場合のみ半角を使用する。<br/>
  例外として、3桁以上の数字に使用するカンマ（ , ）、階層構造を表す際の区切り符（ / ，>）、変数を表す波括弧（{ }）は半角を使用する。<br/>
  記述記号とは、以下を指す。
  - 区切り符　・（中点、ナカグロ）、｜（バーティカルバー、パイプ）、：（コロン）、／（スラッシュ）
  - 句読点　、（読点）、。（句点）
  - 括弧類　（ ）、「 」 、［ ］、【 】
  - つなぎ符　― （ダッシュ）、…（三点リーダー）
  - ？（疑問符）、！（感嘆符）
- 絵文字はUIには基本的に使用しない<br/>
  技術的制約、環境の違い、ローカライズなどの影響を受けるため絵文字の使用は避ける<br/>
  飾りで絵文字を使いたい場合はアイコンの使用を検討する（イメージのすり合わせにアイコンのプロトタイプとして絵文字を参照するのはよい）
- 三点リーダー（…）は基本的に使用しない
- 3桁以上の数値には、カンマ（ , ）を入れる
- 格助詞をむやみに省略しない <br/>
  言葉同士の関係がわかりやすいように、「が」「の」「を」「に」「へ」「と」「より」「から」「で」などの格助詞は省略せずに記載しましょう。
- 条件を示す「場合」「とき」の使い分け<br/>
  「場合」と「とき」に大きな違いはありません。1つの文章に、大前提の条件、小前提の条件がある場合は、大前提に「場合」を、小前提に「とき」を使用します。
